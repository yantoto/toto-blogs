一.什么是DOCTYPE

这是HTML的文档声明，通过这一标签告诉浏览器使用哪一个HTML的标准解析文档。

二.HTML语义化的理解

用正确的标签做正确的事。规定标签的形式，有更可读性的页面。

优点：

  1. 在没有css的情况下也能看清页面结构；
  2. 有利于SEO和搜索引擎简历良好沟通，有助于爬虫抓取更多有效新信息；
  3. 有利于团队维护。

三.src和href区别

都用来引入外部资源。

src：img,video,audio,script。

通过src请求外部资源时，请求的资源会镶嵌到标签所在位置。并且，请求src资源时会停止暂停其他资源下载，直到该资源加载编译执行完毕。

href: a,link。

超链接。浏览器会并行下载资源，不会停止对当前文档的处理。

六.ifame

嵌入式框架，可以把网页的框架和内容嵌入到现有的网页中。

优点：

  1. 处理加载缓慢的内容
  2. 页面上可以独立的显示一个页面或者内容，不与其他元素冲突
  3. 多个页面中重用同一个页面，减少代码冗余
  4. 加载异步，页面可以不等待iframe进行展示
  5. 方便地实现跨域访问

缺点：

  1. 影响主页面onload事件；
  2. 与主页面共享连接池，而浏览器对相同域的连接有限制，所以会影响页面的并行加载；
  3. 页面有多个页面，不易维护；
  4. 浏览器后退按钮没作用；
  5. 无法被一些搜索引擎识别。

连接池概述

连接池（Connection Pool）是一种用于管理数据库连接的缓存技术。它预先创建并维护了一定数量的数据库连接，以供应用程序在需要时直接获取，而不是每次都重新创建连接。这种方式可以显著提高应用程序的性能和效率，特别是在处理大量数据库请求时。

在Web开发中，连接池通常用于管理应用程序与数据库之间的连接。通过使用连接池，应用程序可以更快地获取数据库连接，从而减少连接创建和销毁的时间开销，提高数据库的响应速度和吞吐量。

iframe与主页面共享连接池

当iframe和主页面共享同一个连接池时，这意味着它们都可以从同一个连接池中获取数据库连接。这种情况通常发生在以下场景中：

  1. 同源策略：如果iframe和主页面来自同一个源（即具有相同的协议、域名和端口），则它们可以共享浏览器的某些资源，包括连接池。然而，需要注意的是，这并不意味着它们可以无条件地共享连接池。实际是否共享还取决于浏览器的实现和应用程序的配置。
  2. 应用程序配置：在某些情况下，应用程序可能会配置一个全局的连接池，供所有页面（包括iframe）使用。这种方式可以确保整个应用程序在数据库连接方面的高效性和一致性。

注意事项

  1. 安全性：由于iframe可以嵌入来自不同源的内容，因此在使用时需要特别注意安全性问题。如果嵌入的页面包含恶意代码，它可能会尝试利用共享的连接池进行攻击。因此，建议仅嵌入可信的站点，并使用适当的安全措施来保护连接池和其他敏感资源。
  2. 性能优化：虽然共享连接池可以提高性能，但也需要根据应用程序的实际情况进行配置和优化。例如，可以根据应用程序的负载和数据库的性能来设置连接池的大小和超时时间等参数。
  3. 跨域问题：如果iframe和主页面来自不同的源，则它们之间的通信将受到浏览器的同源策略限制。这可能会导致无法直接共享连接池或其他资源。在这种情况下，需要使用跨域通信技术（如CORS、postMessage等）来实现不同源之间的通信和数据共享。

八.浏览器乱码

编码格式不匹配（如网页编码格式为UTF-8而浏览器编码格式为GB2312），代码编写错误，字体缺失

文档声明和字符编码的区别：

  • 功能不同：文档声明用于指定文档的类型、版本和字符编码，而编码格式则专注于字符的二进制表示方式。
  • 位置不同：文档声明通常位于HTML或XML文档的最前面如，而编码格式则通过<meta>标签在文档的<head>部分指定。
  • 作用范围不同：文档声明影响整个文档的解析和渲染，而编码格式则主要影响文档中字符的显示。

九.跨域

跨域

跨域是浏览器拦截的行为，请求已经发送到后端，后端返回的响应数据被浏览器拦截住了，这就是跨域的流程。跨域是指浏览器允许向服务器发送跨域请求，从而克服Ajax只能同源使用的限制。

同源策略

同源策略是一种约定，由Netscape公司1995年引入浏览器，它是浏览器最核心也最基本的安全功能，如果缺少了同源策略，浏览器很容易受到XSS、CSFR等攻击。所谓同源是指**”协议+域名+端口”三者相同**，即便两个不同的域名指向同一个ip地址，也非同源。

同源策略限制以下几种行为：

  • Cookie、LocalStorage 和 IndexDB 无法读取
  • DOM和JS对象无法获得
  • AJAX 请求不能发送

十二.script、script async 和 script defer

script:解析到该标签，html解析中断，脚本被提取并立即执行。执行结束后，html继续解析。

script async：解析到标签，与html并行解析，有可能在html解析完成之前就执行。

script defer【常用】：解析到标签，与html并行解析，且必须在html解析完后执行。
