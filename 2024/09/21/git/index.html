git

一.创建仓库

  • git init 本地创建

  • git clone 远程服务器上克隆已存在的仓库

git init

直接在当前目录下创建新仓库

git init xx

创建xx文件夹，并在xx文件夹下创建仓库

二.git的三个区域

工作区、暂存区、本地仓库

三.文件状态和提交

  • git status 查看当前文件状态，有哪些文件

  • git add . 将当前目录所有文件提交到暂存区，暂存

  • git rm 可以把暂存区的文件释放

  • git commit –m “第一次提交” 把暂存区的文件提交到本地仓库，称为一次提交

  • git log 查看仓库提交记录(可以查看版本id)

  • git add . 和git commit -m [描述]可写为：

  • git commit -am [描述]

四.回退版本

git reset

git reset – soft [版本id]

谨慎使用hard，如果发生了误删，可以使用：

git reflog 查看操作的历史记录，找到误操作的版本号，使用git reset [该版本号]返回之前的版本（这里理解为套娃）。

注意，只有提交到仓库的数据是可以回溯的，本次修改还没提交的内容会丢失。

五.查看差异

git diff

查看工作区、暂存区和本地仓库之间的差异

  • git diff 默认显示工作区和暂存区的差异

  • git diff HEAD 显示工作区和版本库的差异

  • git diff –cache 显示暂存区和版本库的差异

  • git diff [版本一] [版本二]显示两版本的差异

  • git diff HEAD~ HEAD 显示上一个版本和当前版本的差异

  • git diff HEAD~3 HEAD 显示上第三个版本和当前版本的差异

  • git diff HEAD~3 HEAD file3.txt显示上第三个版本和当前版本的 file3.txt差异

六.删除文件

要删除版本库中的文件，有两种方法。

一：在工作区删除该文件，再提交到暂存区吗，再提交到仓库

二：使用git rm，此时工作区和暂存区都删除了该文件，再commit 提交到仓库。

git rm

  • git rm file2.txt

  • git commit -m “删除了file2.txt”

七.需要忽略的文件

把需要的忽略的文件名放在 .gitignore

echo access.log > .gitignore

八.远程仓库

  • HTTPS:push到远程仓库时需要验证用户名和密码。

  • SSH:不需要用户名和密码，但需要配置公钥。（推荐）

配置秘钥：

  1. 回到根目录
  2. 进入.ssh文件（找不到就自己创建）
  3. 输入ssh-key generate 生成秘钥
  4. 回车，会自动生成两个文件，没有后缀的是私钥文件 以pub结尾的是公钥文件
  5. 打开公钥文件，复制内容，打开github,新建公钥，粘贴内容

本地仓库和本地仓库的联系：

  • push origin origin main:main 将远程main分支和本地main分支关联起来

  • git pull <远程仓库名> <远程分支名>：<本地分支名>

九.文件状态

十.创建分支

  • git branch dev 创建dev分支但并未切换
  • git switch dev 切换到dev分支

这俩等同于

  • git checkout -b dev

合并分支

1.merge(公共)

  • git switch main切换到目标分值

  • git merge dev 将dev分支合并到当前分支main中

2.rebase（个人）

在任意分支都可以进行合并，但合并顺序不同，找到分叉点，并把当前分支后面切换到要合并的分支后面

  • git switch dev
  • git rebase main

查看分支图

  • git log –graph –oneline –decorate –all

删除分支

  • git branch -d dev

十一.合并冲突

  • git diff 查看差异
  • 手动解决冲突
  • git add .
  • git commit -m [描述]
