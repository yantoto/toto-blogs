首先，在ts中定义变量的类型是必不可缺的。

基本数据类型直接在后先跟对应类型即可。

而复杂数据类型往往需要

我们经常会看到两种复杂类型的定义。

interface

这种接口类型，往往定义一个interface.ts，定义接口后export出去。

在要接受的界面，则通过

来接受，注意前面加上type。

使用 interface 的情况

  1. 面向对象编程：

    • 当你需要定义一个对象的结构时，特别是当这个结构可能会被多个类实现或扩展时。

  2. 接口继承：

    • 当你需要创建接口的层次结构，即一个接口继承另一个接口时。

  3. 可选属性和默认值：

    • 当你需要在接口中定义可选属性或默认值时。

  4. 类型检查和实现：

    • 当你希望确保某些对象符合特定的结构时，可以使用 interface 进行类型检查。

使用 type 的情况

  1. 基本类型和联合类型：

    • 当你需要定义基本类型、联合类型或元组时。

  2. 交叉类型：

    • 当你需要组合多个类型以创建新类型时。

  3. 映射类型：

    • 当你需要对现有类型进行转换和修改时。

  4. 函数类型：

    • 当你需要定义复杂的函数签名或重载时。

  5. 别名：

    • 当你需要为已有的类型起一个更有意义的名字时。

总结

  • 使用 interface：当你需要描述对象的结构和行为，特别是在面向对象编程场景下，或者需要可选属性和默认值时。
  • 使用 type：当你需要定义基本类型、联合类型、交叉类型、映射类型，或者为复杂类型起别名时。

选择使用哪一个取决于你的具体需求和代码风格偏好。理解它们的区别和适用场景可以帮助你编写出更加清晰和可维护的 TypeScript 代码。

所以我是认为interface使用地方更多。一般要拼接、交叉类型、起别名的时候则会使用type。
