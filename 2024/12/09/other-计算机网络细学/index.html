两个路由器直接相连的接口处，可指明也可不指明 IP 地址。
如指明 IP 地址，则这一段连线就构成了一种只包含一段线路的特殊“网络” 。这种网络仅需两个 IP 地址，可以使用 /31 地址块。主机号可以是 0 或 1。

ARP作用：根据IP地址找到MAC地址。只能在一个网络上使用，所以有路由的情况无法使用ARP。先广播询问，再单播回复。

ARP四种典型情况：

凡是隔网络，那么寻找本网络的路由器的硬件地址，交给路由器完成。

凡是同网络，直接找硬件地址。

  1. 发送方是主机，要把 IP 数据报发送到本网络上的另一个主机。这时用 ARP 找到目的主机的硬件地址。
  2. 发送方是主机，要把 IP 数据报发送到另一个网络上的一个主机。这时用 ARP 找到本网络上的一个路由器的硬件地址。剩下的工作由这个路由器来完成。
  3. 发送方是路由器，要把 IP 数据报转发到本网络上的一个主机。这时用 ARP 找到目的主机的硬件地址。
  4. 发送方是路由器，要把 IP 数据报转发到另一个网络上的一个主机。这时用 ARP 找到本网络上另一个路由器的硬件地址。剩下的工作由这个路由器来完成。

划分子网和构建超网

为什么子网数是2？

对于子网掩码255.255.192.0，它借用了原本属于主机位的2位（从右往左数的第17位和第18位）作为网络位。这2位的二进制表示可以是00、01、10或11，因此可以划分出4种不同的组合（即2^2=4种）。然而，在子网划分中，通常不会使用全0和全1的组合（它们分别代表整个网络和广播地址），因此实际可用的子网数是4-2=2个。

  • 子网数 = 2^（借用的主机位数）- 2
  • 子网数 = 2^2 - 2 = 4 - 2 = 2

CIDR无类别域间路由

128.14.32.0/20 /20代表什么

代表子网掩码的表示方法

  • 在 IP 地址表示中，“/20” 是一种无类别域间路由（CIDR）的表示方式。它表示子网掩码中从左边开始连续的 “1” 的个数。
  • 对于一个 IPv4 地址，子网掩码长度为 32 位。在 “/20” 的情况下，子网掩码的前 20 位是 “1”，后 12 位是 “0”。
