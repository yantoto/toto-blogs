RSC 是 React Server Components 的缩写，它是 React 团队推出的一种新的组件模型，旨在改善 React 应用的性能和开发体验。RSC 允许开发者在服务器端渲染组件，并将渲染结果发送到客户端，从而减少客户端的 JavaScript 加载量和计算负担。

RSC 的核心特点

  1. 服务器端渲染：

    • RSC 在服务器端渲染组件，生成静态的 HTML 或 JSON 数据，然后发送到客户端。
    • 客户端只需要渲染交互部分，减少了客户端的计算负担。

  2. 零客户端 JavaScript：

    • RSC 组件默认不会向客户端发送 JavaScript 代码，因此不会增加客户端的包体积。
    • 这对于静态内容或不需要交互的组件非常有用。

  3. 与客户端组件共存：

    • RSC 可以与传统的客户端组件（Client Components）无缝协作。
    • 客户端组件仍然可以处理交互逻辑，而 RSC 负责渲染静态内容。

  4. 按需加载：

    • RSC 支持按需加载组件，只有在需要时才会从服务器获取组件内容。
    • 这可以显著减少初始加载时间。

  5. 数据获取与组件渲染一体化：

    • RSC 允许在组件内部直接获取数据，而不需要额外的 API 调用。
    • 数据获取和组件渲染可以在服务器端一次性完成。

RSC 的工作原理

  1. 服务器端渲染：

    • 当用户请求页面时，服务器会根据路由和组件树渲染 RSC。
    • 渲染结果是一个 JSON 格式的描述（称为 “React Flight” 协议），而不是传统的 HTML。

  2. 客户端接收与渲染：

    • 客户端接收到服务器发送的 JSON 数据后，React 会将其转换为实际的 DOM 元素。
    • 如果组件需要交互，客户端会加载对应的客户端组件。

  3. 混合渲染：

    • RSC 和客户端组件可以混合使用。例如，一个页面的静态部分可以用 RSC 渲染，而交互部分可以用客户端组件渲染。

RSC 的优势

  1. 更小的客户端包体积：

    • 由于 RSC 组件不会向客户端发送 JavaScript 代码，客户端的包体积会显著减小。

  2. 更快的页面加载：

    • 服务器端渲染可以减少客户端的计算负担，从而加快页面加载速度。

  3. 更好的 SEO：

    • 由于内容是在服务器端渲染的，搜索引擎可以更好地抓取和索引页面内容。

  4. 简化数据获取：

    • 数据获取和组件渲染可以在服务器端一次性完成，减少了客户端的复杂性。

RSC 的局限性

  1. 不支持客户端交互：

    • RSC 组件本身不支持客户端交互（如事件处理、状态管理等），需要与客户端组件配合使用。

  2. 需要服务器支持：

    • RSC 依赖于服务器端渲染，因此需要服务器环境支持。

  3. 学习曲线：

    • RSC 是一种新的开发模式，开发者需要学习如何正确使用它。

总结

RSC 是 React 的一种新特性，旨在通过服务器端渲染和按需加载来优化性能和开发体验。它特别适合用于渲染静态内容或数据密集型页面，同时可以与客户端组件无缝协作。随着 React 生态的发展，RSC 可能会成为构建高性能 React 应用的重要工具。
