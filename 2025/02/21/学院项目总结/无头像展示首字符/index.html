提问

几步就写完了，但是，为什么同一用户背景一直是同一颜色？哪一步让随机颜色被分类了？

先讲讲生成函数里的逻辑：

  1. 获取颜色对象里的键
  2. 将传入数据（名字里的第一个字符）转换为数组，再进行累加生成哈希值
  3. 哈希值余上颜色的长度，获取索引
  4. 返回该索引对应的值

综上，这个函数的核心逻辑是根据传入的字符串 name 计算一个哈希值，然后使用这个哈希值从 colorMap 对象中随机选取一个渐变色返回。由于相同的 name 会计算出相同的哈希值，因此对于相同的 name 输入，函数会始终返回相同的渐变色。

这种方法的好处是生成的颜色会在规定的种类中，不会出现背景和文字颜色太相似导致难以看清；不足之处在于生成的颜色需要自己定义，无法生成除了定义以外的其他颜色。
