子组件只负责交互和UI,不负责业务逻辑。

所以，子组件需要接收父组件的数据和事件函数，在子组件中展示数据并绑定对应的事件函数。

可以设计一些剩余参数，将少数情况的页面展示出来。

比如子组件中有点赞，那么只需要接收父组件的 是否已点赞 点赞数量 点赞触发的函数

并把这些展示出来，父组件在传入的函数内部来写调用的后端接口，子组件只需把函数触发（顶多把当前事件对象传进去，一般是不会考虑传入什么id/数量什么的，不涉及业务逻辑）。

这个界面的点赞，应该是在父组件的传入函数中写上调用接口，子组件只需在点击事件中调用该函数即可。

如果要实现分页（另一个场景），那么子组件中页数改变，会调用父组件的函数将页数信息传回，传参只涉及通用的组件逻辑，而不涉及具体场景逻辑。
