那么，为什么要使用scss？又为什么要用这种引入方式？

原因：

  1. 模块化：通过模块化的方式引入样式，可以确保每个组件的样式是独立的，不会与其他组件的样式发生冲突。这在大型项目中尤为重要，因为不同组件可能会使用相同的类名，如果不进行模块化处理，这些类名可能会相互覆盖。

  2. 作用域隔离：使用 CSS Modules（即 .module.scss 文件），可以为每个组件生成唯一的类名，从而避免全局样式污染。这意味着你可以放心地使用通用类名而不必担心它们会意外地影响其他组件。

  3. 编译支持：.scss 文件需要经过预处理器（如 Sass）的编译才能转换为标准的 CSS。通过这种方式引入，构建工具（如 Webpack、Next.js 等）会自动处理这些文件，将其编译为浏览器可以理解的 CSS。

  4. 动态绑定：引入的 styles 对象是一个 JavaScript 对象，其中键是类名，值是编译后的类名。这使得你可以在 JSX 中动态绑定类名，例如：

    这样可以确保类名是唯一且正确的。

  5. 热更新：在开发环境中，CSS Modules 通常支持热更新功能，这意味着当你修改 .scss 文件时，样式会立即反映在页面上，而无需手动刷新整个页面。
