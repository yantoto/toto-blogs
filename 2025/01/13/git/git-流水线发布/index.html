流水线发布流程

拉取代码后，现在自己的分支上开发，注意每天至少提交一次；

代码无误后，看自己要发的测试环境；如果自己的功能只有一个，且测试环境上没有比自己超前或者落后的代码，则可以直接在dev或者qa上跑自己的分支；

如果测试环境上的代码与自己的有差异（比如别人新添的功能还在测试，没有合在master上，而我的代码是从master拉取的），那么不能直接在测试流水线上跑自己的分支，有可能冲掉别人的代码或者被别人冲掉；

所以有差异的情况下，需要将自己的代码合在dev或者qa分支上，再用dev或者qa分支跑各自流水线，两个流水线保持清爽，互不影响。

总结一下，就是测试环境下既要上传自己的，也要保留他人的，如果自己的分支比较落后，没有同步他人新代码，那么则需要各自把代码合在同一份分支上，再去跑流水线。不要冲他人或者被他人冲掉！！！！！
